1. Choose a website

You can choose any HTML website that you want to parse or cache, even with improper HTML formatting.

2. Setup website in daisyfetch

You can use CSS3 selectors you are used to work with and fetch only those parts of website that you are really interested in. Moreover, daisyfetch will cache results or notify you on your specified endpoint, if website was changed.

3. Fetch website as a JSON

All responses have CORS setted to *, so you can fetch your resource from JavaScript. You can use HEAD request to see if data has been changed or not.

About application

Robust JEE7 backend on Wildfly

Daisyfetch is deployed on Openshift and is running on Wildfly application server. The data layer is serviced by embedded infinispan cache with passivation onto file system.

Dynamical UI with AngularJS

Presentation layer of application is implemented in AngularJS, which is communicating with JEE7 backend via REST-full interface.

Picketlink for login

There is no need for registration, you can just use your existing login from Facebook or Google. This is accomplished with picketlink library.
